# Security Policy

## 🔒 Security Overview

JobTailor takes security seriously. As a job search application that handles user data and API keys, we are committed to ensuring the security and privacy of our users.

## 🚨 Reporting Security Vulnerabilities

If you discover a security vulnerability in JobTailor, please help us by reporting it responsibly.

### How to Report

**Please DO NOT report security vulnerabilities through public GitHub issues.**

Instead, please report security vulnerabilities by emailing: [Your Email Address]

Include the following information in your report:
- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact and severity
- Any suggested fixes or mitigations

### What to Expect

- **Acknowledgment**: We'll acknowledge receipt of your report within 48 hours
- **Investigation**: We'll investigate and work on a fix
- **Updates**: We'll keep you informed about our progress
- **Resolution**: We'll notify you when the issue is resolved
- **Credit**: With your permission, we'll credit you in our security advisory

## 🛡️ Security Measures

JobTailor implements several security measures:

- **Client-side API key storage**: Keys are stored locally in browser localStorage
- **No server-side data collection**: All processing happens client-side
- **HTTPS-only communication**: All API calls use secure HTTPS connections
- **Input validation**: User inputs are validated and sanitized
- **Dependency monitoring**: Regular security updates via Dependabot

## 📋 Responsible Disclosure

We kindly ask that you:
- Give us reasonable time to fix the issue before public disclosure
- Avoid accessing or modifying user data
- Don't perform DoS attacks or degrade service performance
- Don't spam our systems with automated vulnerability scanners

## 🙏 Recognition

We appreciate security researchers who help keep JobTailor safe. With your permission, we'll acknowledge your contribution in our security advisories.

Thank you for helping keep JobTailor and its users secure! 🛡️
